# 数字分身 - Digital Twin

## 目标

系统完全学习创作者的思维模式，可以代替创作者进行初步的选题筛选。

## 构建流程

```
学习阶段 → 特征提取 → 分身训练 → 筛选测试 → 正式启用
```

## 学习数据来源

| 数据类型 | 内容 |
|----------|------|
| 选题历史 | 用户选择过的所有命题 |
| 标题偏好 | 用户从候选中选择了哪些标题 |
| 改词记录 | 用户如何修改生成的标题 |
| 反馈数据 | 用户对最终标题的评分 |
| 身份定位 | 用户配置的 role/mission/tone |

## 学习特征

### 1. 命题特征

| 特征 | 说明 |
|------|------|
| 命题复杂度 | 用户倾向于处理简单还是复杂的命题 |
| 命题方向 | 用户更关注职场/技术/生活哪个领域 |
| 命题深度 | 用户喜欢宏观分析还是微观场景 |

### 2. 标题特征

| 特征 | 说明 |
|------|------|
| 维度偏好 | 用户更倾向选择哪个维度 |
| 风格偏好 | 用户喜欢理性/感性/幽默哪个风格 |
| 长度偏好 | 用户倾向于短标题还是长标题 |
| 结构偏好 | 用户喜欢"为什么"还是"如何"开头 |

### 3. 行为特征

| 特征 | 说明 |
|------|------|
| 采纳率 | 用户使用生成标题的比例 |
| 修改模式 | 用户修改标题的常见方式 |
| 拒绝原因 | 用户拒绝标题的常见理由 |

## 验收标准

- 数字分身匹配度 > 80%
- 筛选结果与创作者实际选择的一致率 > 75%
- 筛选速度：每次筛选 < 5 秒

## 安全约束

1. **最终决策权归用户**：数字分身只做初筛，最终决策仍由用户做出
2. **透明化**：用户可查看分身为什么筛选了某个标题
3. **可禁用**：用户可随时关闭数字分身功能
4. **定期校准**：每 50 次筛选后，询问用户是否准确并更新模型
