# 知识拓扑图谱 - Knowledge Topology

## 目标

提取高频实体关系，发现认知盲区，标注"过度开发区"和"未触及区"。

## 构建流程

```
历史选题 → 实体提取 → 关系抽取 → 图谱构建 → 区域标注
```

## 实体类型

| 类型 | 示例 |
|------|------|
| 人物 | 程序员、职场人、老板、学生 |
| 概念 | AI、认知、努力、教育 |
| 场景 | 职场、学习、生活、社交 |
| 情绪 | 焦虑、困惑、恐惧、兴奋 |
| 行动 | 学习、转型、提升、改变 |

## 关系类型

| 类型 | 示例 |
|------|------|
| 因果关系 | AI → 失业 |
| 归属关系 | 程序员 → 职场 |
| 对立关系 | 努力 → 贫穷 |
| 支撑关系 | 认知 → 成长 |

## Prompt - 实体关系提取

```
你是知识图谱构建专家。从选题中提取实体和关系。

选题列表：{{TOPIC_LIST}}

提取内容：
1. 实体列表（人物、概念、场景、情绪、行动）
2. 关系列表（因果、归属、对立、支撑）
3. 出现频次

返回 JSON：
{
  "entities": [
    {"name": "程序员", "type": "人物", "frequency": 15},
    {"name": "AI", "type": "概念", "frequency": 23},
    {"name": "焦虑", "type": "情绪", "frequency": 8}
  ],
  "relations": [
    {"from": "AI", "to": "失业", "type": "因果", "frequency": 12},
    {"from": "程序员", "to": "焦虑", "type": "归属", "frequency": 7}
  ]
}
```

## Prompt - 区域标注

```
你是认知地图分析师。基于实体关系图谱，标注认知开发区域。

实体统计：{{ENTITIES}}
关系统计：{{RELATIONS}}

定义：
- 过度开发区：该实体/关系出现频率过高，可能导致思维定势
- 未触及区：该实体/关系从未或很少出现，可能是认知盲区
- 平衡区：频率适中，认知健康

返回 JSON：
{
  "over_explored": [
    {"entity": "AI", "reason": "出现频率 23 次，建议降低相关选题比例", "suggestion": "探索其他技术领域"},
    {"entity": "努力", "reason": "过度讨论反努力话题", "suggestion": "转向建设性话题"}
  ],
  "under_explored": [
    {"entity": "家庭", "reason": "从未出现", "suggestion": "可作为新选题方向"},
    {"entity": "长期规划", "reason": "出现频率 < 2", "suggestion": "探索时间维度话题"}
  ],
  "balance_zones": [
    {"entity": "教育", "frequency": 5, "status": "健康"}
  ]
}
```

## 验收标准

- 知识图谱覆盖率 > 80%
- 区域标注准确率 > 75%
- 建议采纳后选题多样性提升 > 30%
