Step 1 — 用户输入

1 用户在终端输入一个明确的主题
命题包含具体对象、冲突元素和事实支撑，熵值 ≥ 2.5，直接放行

Step 2 — Phase 0: 意图识别

2 classify_intent() — 判断是否触发 PRISM-OS
检测用户输入是否包含写作/选题关键词，或疑问句式。包含"写文章"、"选题"、"如何写"等触发词时启动。
trigger: true ← "选题"关键词存在 confidence: 0.65 reason: "包含选题关键词"
✓ 触发 PRISM-OS，进入 Phase 1

Step 3 — Phase 1: 苏格拉底网关

3 socratic_gateway() — 熵值计算 + 决策（Phase 4.7 优化）
Phase 4.7 变更：熵值计算改为纯公式实现，0 次 LLM 调用。

三个维度打分（规则版）：
Object_Clarity（对象清晰度）：正则匹配具体对象/模糊对象/无对象
Conflict_Tension（冲突张力）：关键词检测（越X越Y/反常识等）
Fact_Support（事实支撑）：数据/案例/现象描述检测
熵值公式：Entropy = Object×0.4 + Conflict×0.4 + Fact×0.2
决策阈值：≥ 2.5 → pass、1.5-2.5 → clarify、< 1.5 → blocked
input_type: "sentence" object_clarity: 0.8 conflict_tension: 0.75 fact_support: 0.7 entropy_score: 2.65 decision: "pass" # LLM 调用：0 次（Phase 4.7 规则版）
✓ 熵值 2.65 ≥ 2.5，直接放行，进入 Phase 2

Step 4 — Phase 1.5: 备选检查

4 check_related_backups() — 查询飞书备选队列
1. 调用 check_expired_backups() 清理 30 天前过期备选
2. 读取飞书表中 类型=备选、状态=备选 的所有记录
3. 计算当前命题与备选的 Embedding 相似度（0.4×Jaccard + 0.6×Cosine）
4. 返回相似度 > 0.7 的记录
# 备选队列查询结果 matched_backups: [] # 无相似备选
无相关备选，继续 Phase 2

Step 5 — Phase 2: 棱镜引擎

5 prism_engine() — 四维标题生成
按四个维度各生成 3 个候选标题（共 12 个）：
reversal（逆向拆解）、micro_scene（微观切片）、systemic_flaw（系统归因）、bridge（认知脚手架）
每个维度有独立 prompt，包含禁用词检查和长度限制（18-28字）
# prism_engine() 返回 status: "success" candidates: [ {title: "程序员失业潮：被AI取代的真相", dimension: "reversal", char_count: 18}, {title: "从某厂裁员看技术人的职场危机", dimension: "micro_scene", char_count: 16}, ... ] orthogonal_count: 10

Step 6 — Phase 2.5: 正交性检查

6 check_orthogonality() — Embedding 相似度过滤
预计算所有标题向量，计算 Cosine 相似度。相似度 > 0.75 的标题标记为非正交。
Embedding API 不可用时降级到纯 Jaccard。
candidates[0].orthogonal: true # max_similarity=0.52 candidates[1].orthogonal: false # max_similarity=0.81 ← 被标记

Step 7 — Phase 3: 现实校验锚

7 reality_anchor() — 搜索查重 + 竞争度评估
1. 调用 Tavily API（优先）或 Firecrawl API（备用）搜索标题
2. 计算与搜索结果的 Jaccard 相似度作为查重率
3. 查重率 < 0.3 → 蓝海，0.3-0.7 → 黄海，> 0.7 → 红海
4. 查重率 > 0.8 → 拒绝该标题
# reality_anchor() 返回 validated: [ {title: "...", duplicate_rate: 0.22, competition_level: "蓝海", novelty_score: 0.78}, ... ] statistics: { blue_ocean: 4, yellow_ocean: 5, red_ocean: 1, avg_novelty: 0.71 }

Step 8 — Phase 3.5: 数字分身筛选

8 digital_twin_filter() — 模拟创作者思维筛选
1. 调用 learn_thinking_pattern() 从 topic_log.yaml 学习思维特征
2. 根据学习到的维度权重、风格关键词进行个性化筛选
3. 优先选择符合创作者偏好的选题，过滤模式化选题
# learn_thinking_pattern() 返回 thinking_pattern: "理性、克制、反常识" dimension_weights: {reversal: 0.35, micro_scene: 0.25, ...} confidence: 0.8 # digital_twin_filter() 返回 selected_topics: [ {topic: "程序员失业潮：被AI取代的真相", selection_reason: "符合逆向拆解偏好"}, ... ]

Step 9 — Phase 4.5: CCOS v2.0 认知推进流大纲

9 cognitive_outline_workflow() — 14项动态认知大纲（Phase 4.7 优化）
Phase 4.7 优化：
1. Layer 2 三个 LLM 调用改为并行化（extract_core_problem / extract_cognitive_tension / infer_potential_directions）
2. 双平台共享 Layer 2 结果（18次→12次 LLM）

CCOS（Content Cognition Outline System）v2.0 替代旧版目录式大纲，8层认知处理：
Layer 0（认知对齐）：七类追问，用户可选跳过

Step 1 — 输入分类

1 classify_input() — 判断输入类型
检测规则：中文 < 10 字符 → keyword；包含问号/疑问词 → question；否则 → sentence
"AI让我焦虑" 共 6 个字符，无问号 → input_type: "keyword"
input_type: "keyword"

Step 2 — 熵值计算

2 calculate_entropy() — 三维度打分
object_clarity：无具体对象，"AI让我焦虑"是情绪表达 → 0.0
conflict_tension：无矛盾冲突，纯情绪 → 0.0
fact_support：无具体事实，纯感受 → 0.0
Entropy = 0.0×0.4 + 0.0×0.4 + 0.0×0.2 = 0.0
object_clarity: 0.0 conflict_tension: 0.0 fact_support: 0.0 entropy_score: 0.0 decision: "clarify" # 1.5 ≤ 0.0 < 2.5 → clarify
⚠ 熵值 0.0，需要追问澄清

Step 3 — 生成追问问题

3 generate_clarification_questions()
keyword 类型追问原则：追问核心观点、受众关联、期望行动
返回 2-3 个追问，帮助用户将模糊意图明确化

Step 4 — 生成选题方向 + 保存备选队列

4 generate_directions() + save_backup()
为 keyword 类型生成 2-3 个具体选题方向，追加到返回结果，不影响原有追问逻辑。
每个方向调用 save_backup() 存入飞书备选队列（类型=备选，状态=备选），相同标题自动去重。

✓ 3 个方向已存入飞书备选队列

5 check_related_backups() — 匹配已有备选
清理过期备选后，读取所有备选队列记录，计算与当前命题的 Embedding 相似度。
假设飞书已有 "AI时代年轻人职业选择" 的备选，相似度 0.73 > 0.7 阈值：

expression_angles：不同类型创作者的切入角度 + 匹配度

Step 1 — 思维特征学习

1 learn_thinking_pattern() — 从历史学习
读取 topic_log.yaml 最近 50 条选题历史，提取命题文本。
将前 10 条作为样本发送给 LLM，分析维度偏好、风格关键词、思维特征。
置信度 = min(1.0, 历史条数 / 20)，20 条以上满置信度。
# 输入：topic_log.yaml 历史记录 [ {thesis: "为什么年轻人不愿意生孩子", dimension: "reversal"}, {thesis: "小县城年轻人的生存困境", dimension: "micro_scene"}, ... ] # learn_thinking_pattern() 返回 thinking_pattern: "理性、克制、反常识、深挖数据" dimension_weights: { reversal: 0.35, micro_scene: 0.30, systemic_flaw: 0.20, bridge: 0.15 } style_keywords: ["为什么", "其实", "数据", "真相"] confidence: 0.75 # 基于 15 条历史

Step 2 — 数字分身筛选

2 digital_twin_filter() — 个性化筛选候选标题
将学习到的思维特征注入 prompt，引导 LLM 模拟创作者偏好：
1. 优先选择符合维度权重的标题（reversal 权重最高 0.35）
2. 过滤使用 style_keywords 中陈词滥调的标题
3. 保留具有"认知落差"的选题
4. 过滤过于模式化的标题
# digital_twin_filter() 输入 candidates: [12个候选标题] thinking_pattern: "理性、克制、反常识、深挖数据" dimension_weights: {reversal: 0.35, micro_scene: 0.30, ...} # 返回 selected_topics: [ { topic: "为什么"努力就能成功"是当今社会最大陷阱？", selection_reason: "符合逆向拆解偏好+反常识特征", confidence: 0.82 } ] rejected_topics: [ { topic: "AI时代必知的5个生存法则", rejection_reason: "过于模式化，方法论标题不符合克制风格", confidence: 0.65 } ]

Step 3 — 用户选择

3 用户从 selected_topics 中选择或拒绝
数字分身推荐 3 个标题，用户可能：
1. 接受其中一个推荐 → match = true
2. 拒绝所有推荐，自行选择其他标题 → match = false
# 场景 A：用户接受了推荐 user_selected: "为什么"努力就能成功"是当今社会最大陷阱？" twin_selected: ["为什么"努力就能成功"...", "程序员失业潮真相", "..."] match: true # 场景 B：用户拒绝所有推荐 user_selected: "AI时代必知的5个生存法则" # 这是 rejected_topics 中的标题 twin_selected: [...] match: false

Step 4 — 反馈记录

4 record_twin_feedback() — 记录到 twin_feedback.yaml
保存反馈：命题 + 分身推荐列表 + 用户实际选择 + 是否匹配。
每 50 次筛选后触发校准提示，建议用户调整思维特征配置。
# twin_feedback.yaml 记录 - thesis: "为什么年轻人不愿意生孩子" twin_selected: ["..."] user_selected: "为什么"努力就能成功"是当今社会最大陷阱？" match: false date: "2026-05-14"

Step 5 — 匹配度计算

5 calculate_twin_accuracy() — 统计匹配率
统计 twin_feedback.yaml 中所有反馈的 match 比率。
阈值策略：先跑几轮数据看基线再定，暂不设硬阈值。
# calculate_twin_accuracy() 返回 total_count: 28 match_count: 17 match_rate: 0.607 average_confidence: 0.71 last_calibration: "2026-05-10" suggestion: "建议关注 reversal 维度，当前匹配率 60.7%"
