# 认知裂缝捕捉 - Cognitive Crack Hunter

## 目标

自动监控全网信息源，发现"社会共识与现实的裂缝"。

## 信息源

| 类型 | 来源 | 监控频率 |
|------|------|----------|
| RSS 订阅 | 用户配置的 RSS 源 | 每日 |
| 社交媒体 | Twitter/X API | 实时 |
| 新闻聚合 | Google News / Bing News | 每小时 |
| 论坛 | Reddit、知乎等 | 每日 |

## 裂缝类型

| 类型 | 示例 |
|------|------|
| 数据裂缝 | 共识认为 X，但数据显示 Y |
| 逻辑裂缝 | 共识基于 A，但 A 已被证伪 |
| 时间裂缝 | 2020 年的共识已不适用 2026 年 |
| 人群裂缝 | 某些群体的经验与主流叙事不符 |

## Prompt - 裂缝识别

```
你是认知裂缝猎人。分析信息源中发现的内容，判断是否存在"社会共识与现实的裂缝"。

信息源：{{SOURCE}}
内容摘要：{{CONTENT_SUMMARY}}
发表时间：{{PUBLISH_DATE}}

裂缝检测维度：
1. 数据裂缝：内容中的数据是否与社会共识矛盾？
2. 逻辑裂缝：内容的论证逻辑是否有漏洞？
3. 时效性裂缝：内容是否揭示了观点的时效性？
4. 人群裂缝：是否揭示了特定群体的不同经历？

返回 JSON：
{
  "has_crack": true/false,
  "crack_type": "数据裂缝/逻辑裂缝/时效性裂缝/人群裂缝/无",
  "consensus": "当前社会共识是什么",
  "reality": "实际情况是什么",
  "gap_description": "裂缝的具体描述",
  "confidence": 0.85,
  "source": "信息来源",
  "suggested_topic": "建议的选题方向"
}
```

## Prompt - 裂缝优先级排序

```
你是认知裂缝分析师。对发现的所有裂缝进行优先级排序。

裂缝列表：{{CRACK_LIST}}

排序维度：
1. 裂缝深度：共识与现实的差距有多大？
2. 受众相关性：与目标受众的兴趣有多相关？
3. 时效性：是否是当前热门话题？
4. 创作空间：是否有足够的角度可以展开？

返回 JSON：
{
  "prioritized_cracks": [
    {
      "rank": 1,
      "crack": {...},
      "priority_score": 0.9,
      "reason": "裂缝深度大且与受众高度相关"
    }
  ]
}
```
