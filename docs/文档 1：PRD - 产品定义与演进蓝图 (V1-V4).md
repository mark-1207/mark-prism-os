# 文档 1：PRD - 产品定义与演进蓝图 (V1-V4)


## 一、产品核心定义

### 1.1 产品定位

PRISM-OS 是一个基于大语言模型的**认知澄清与内容战略引擎**。它不是标题生成器，而是通过解构用户意图、对接外部知识库、执行逻辑审计，输出具备“认知落差”的内容资产。

### 1.2 核心差异化

市场上的选题工具分两类：

- **“给你新知识”型**：依赖信息差，护城河越来越低

- **“给你方法论”型**：依赖执行力，读者看完觉得有用但做不到

PRISM-OS 是第三类：

- **“帮你拆掉旧框架”型**：不给新信息，不给步骤清单，而是让用户意识到自己一直在用一把错误的尺子量世界

### 1.3 北极星指标

- **认知落差 (Cognitive Gap)：** > 50%（用户看到标题后的“这就是我想表达却说不出来的”比例）

- **素材就绪度 (Readiness)：** > 80%（生成标题后，用户现有素材能支撑内容创作的比例）

- **逻辑严密度 (Logic Integrity)：** > 90%（标题经过逻辑审计后无明显谬误的比例）

## 二、目标用户画像

### 2.1 核心用户

- **人生阶段：** 事业上升期的从业者/小企业主（28-45 岁）

- **核心焦虑：** 不想成为时代弃儿，担心错过 AI 红利期

- **核心渴望：** 通过 AI 实践提升赚钱能力，实现财富增长

- **找不到答案的问题：**

  - 如何把 AI 工具真正用到赚钱上，而不是停留在“会用”层面

  - 认知提升到底怎么转化成实际收益

  - 别人靠 AI 赚到钱了，我为什么还在原地

### 2.2 创作者定位

- **身份：** 揭秘者 + 框架型思考者

- **语调：** 理性、克制、反常识

- **核心价值：** 拆掉旧框架，建立新视角

## 三、全生命周期演进路线图

### V1.0：逻辑闭环 (MVP)

**发布时间：** 已完成\
**核心目标：** 实现从模糊意图到高质标题的转化

**核心功能：**

1. **苏格拉底网关 (Socratic Gateway)**

   - 执行“信息熵”检查

   - 强制用户从“情绪”转向“命题”

   - 三级响应：直接通过/迫选追问/拦截重构

2. **棱镜引擎 (Prism Engine)**

   - 四维正交生成：

     - **逆向拆解 (Reversal)：** 攻击世俗共识

     - **微观切片 (Micro-Scene)：** 极致具体的反常场景

     - **系统归因 (Systemic Flaw)：** 结构性、系统性的崩坏

     - **认知脚手架 (Bridge)：** 从旧到新的过渡桥梁

3. **现实校验锚 (Reality Anchor)**

   - 搜索引擎 API 查重

   - 标注蓝海/黄海/红海

   - 不删除任何标题，只标注现实情况

**存储方案：**

- 本地 JSON/YAML 文件

- 用户配置、选题日志、词汇指纹

**验收标准：**

- 初始化流程可用性 > 90%

- 网关拦截准确率 > 85%

- 四维标题正交率 > 75%

- 北极星达成率 > 60%

### V1.5：执行下沉 (RAG & Gap Analysis)

**预计发布：** 累计 20 次选题记录后\
**核心目标：** 解决“有好标题但写不出来”的问题

**核心功能：**

1. **知识网关 (Knowledge Gateway)**

   - 接入外部知识库（YouMind API / 本地 Markdown / Notion）

   - 支持用户自定义数据源

   - 实现 RAG（检索增强生成）

2. **素材缺口分析 (Gap Analysis)**

   - 提取标题所需的“证据链 (Evidence Chain)”

   - 对比知识网关返回的素材

   - 输出结构化的 `Gap_Report`：

     - 现有支撑点

     - 逻辑断层

     - 建议补充的素材类型

3. **双端大纲生成器 (Cross-Platform Transformer)**

   > **V1.5 已升级为 CCOS v2.0（Phase 4.5）**

   **Phase 4.5：CCOS v2.0 认知推进流大纲**

   - **Layer 0 认知对齐：** 七类追问，人机协同收敛写作方向（方向/立场/情绪/案例/反直觉/读者画像/边界）
   - **Layer 1-3 选题解析：** 选题类型 / 核心问题 / 认知张力 / 主结构 / 推进方式
   - **Layer 4 认知模块编排：** HOOK / CASE / EXPLAIN / MODEL / COUNTER / ACTION / BOUNDARY 模块流
   - **Layer 7 作者性注入：** 整合数字分身思维特征，注入认知倾向/表达气质/价值倾向/长期母题
   - **Layer 8 内容势能设计：** 张力曲线 / 情绪曲线 / 认知落差 / 节奏变化
   - **14 项动态认知大纲输出：** 内容目标 / 用户动机 / 核心认知冲突 / 内容立场 / 作者性设定 / 主结构 / 推进方式 / 认知模块流 / 势能曲线 / 案例插入点 / 信息密度要求 / 语言风格 / Anti-AI要求 / 最终动态认知大纲

   **Phase 4.6：Gap Analysis 素材就绪度**

   - 扫描 Obsidian 知识库
   - 综合就绪度 < 0.3 → 中断；0.3-0.6 → 警告；>= 0.6 → 通过
   - 输出 missing_evidence 清单

   **公众号/小红书结构选择：** 由 Layer 0 用户讨论结果（内容目标 + 情绪取向）动态决定，不写死。具体映射规则存储在 `ccos_settings.yaml`，包括四大主结构 × 8种内容目标 × 6种推进方式的推荐关系，以及平台权重差异配置。

   **公众号示例侧重：** 认知升级型 × 递进推进，势能侧重认知落差（当用户内容目标=认知升级时）
   **小红书示例侧重：** 故事驱动型 × 情绪推进，势能侧重情绪曲线（当用户内容目标=情绪共鸣时）

   > 具体映射关系见 `skills/prism-os/data/ccos_settings.yaml`，可动态调整无需改代码。

**验收标准：**

- 素材就绪度评估准确率 > 75%

- 缺口识别精准度 > 70%

- 双端大纲可用性 > 80%

### V2.0：战略逻辑 (Logic Audit & Journey)

**预计发布：** 累计 50 次选题记录后\
**核心目标：** 从单点突破走向系列化、防杠化

**核心功能：**

1. **逻辑压力测试 (Stress Test)**

   - **替代多 Agent 对抗**

   - 调用形式逻辑检查器

   - 识别命题中的：

     - 循环论证

     - 幸存者偏差

     - 因果倒置

     - 稻草人谬误

2. **认知旅程规划 (Cognitive Journey Mapping)**

   - 计算当前选题与历史选题的语义距离

   - 确保认知深度是线性递进而非原地打转

   - 生成“前后衔接”的选题矩阵

3. **受众语言指纹动态更新**

   - 深度分析受众的评论反馈（如果有）

   - 提取他们高频使用的词汇

   - 反向注入到标题生成逻辑中

**验收标准：**

- 逻辑谬误识别率 > 80%

- 认知旅程连贯性 > 85%

- 受众语言匹配度 > 70%

---

### V3.0：自主进化 (Self-Evolution)

**预计发布：** 累计 20 篇真实发布数据后\
**核心目标：** 对抗创作者的思维定势

**核心功能：**

1. **刺客机制 (Assassin Mechanism)**

   - 定期扫描 3 个月前的爆款日志

   - 由系统发起“逻辑反转”

   - 强制创作者否定旧观点

   - 实现认知迭代

2. **知识拓扑图谱 (Knowledge Topology)**

   - 提取高频实体关系

   - 发现认知盲区

   - 标注“过度开发区”和“未触及区”

3. **Prompt 自动变异 (Prompt Evolution)**

   - 根据用户“改词记录”和“维度偏好”

   - 自动修改系统的 `System_Instruction`

   - 实现真正的自我优化

**验收标准：**

- 刺客机制触发准确率 > 75%

- 知识图谱覆盖率 > 80%

- Prompt 变异有效性 > 70%

---

### V4.0：全域感知 (Matrix & Digital Twin)

**预计发布：** V3 运行 3 个月后\
**核心目标：** 成为创作者的数字副脑

**核心功能：**

1. **认知裂缝捕捉 (Cognitive Crack Hunter)**

   - 自动监控全网信息源（RSS/Twitter/特定 API）

   - 对比本地词汇指纹库

   - 主动发现“社会共识与现实的裂缝”

2. **主动选题推送**

   - 每日自动推送到创作者终端

   - 格式：“今天发生了 X，大部分人认为是 Y，但基于你的知识库，本质是 Z，建议写一篇……”

3. **数字分身 (Digital Twin)**

   - 系统完全学习创作者的思维模式

   - 可以代替创作者进行初步的选题筛选

   - 创作者只需要对最终候选进行决策

**验收标准：**

- 裂缝识别准确率 > 70%

- 主动推送有效率 > 60%

- 数字分身匹配度 > 80%

## 四、长期愿景

PRISM-OS 的终极目标是成为一个**越用越智能、人的参与度越来越低**的系统。最终系统能够：

- 自主发现认知靶子

- 自动维护语言指纹库

- 构建选题拓扑图谱

- 自我校准和进化

**但始终保持一个原则：** 系统是创作者的“副脑”，而不是替代品。最终的决策权和创作权永远属于人类。
