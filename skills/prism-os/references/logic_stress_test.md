# 逻辑压力测试 - Logic Stress Test

## 检测的逻辑谬误类型

| 谬误类型 | 定义 | 示例 |
|----------|------|------|
| 循环论证 | 结论即前提 | "为什么成功的人都很努力？" → 用"成功"定义"努力" |
| 幸存者偏差 | 只关注成功案例 | "为什么所有富豪都早起？" → 忽略了失败案例 |
| 因果倒置 | 混淆因果关系 | "为什么聪明人都读书多？" → 可能是读书让人聪明 |
| 滑坡谬误 | 夸大连锁反应 | "AI 会让所有人失业" → 过度推论 |

## 与现实校验锚的关系

逻辑压力测试与 Phase 3（现实校验锚）**并行执行**，不串行等待。

- 现实校验锚：基于外部搜索数据验证新颖度
- 逻辑压力测试：基于内部推理检查逻辑漏洞

两者结果合并，共同决定最终输出。

## Prompt

```
你是逻辑审计员。检测标题中的逻辑谬误。

待检测标题：{{TITLE}}

检测项：
1. 循环论证：结论即前提
2. 幸存者偏差：只关注成功案例
3. 因果倒置：混淆因果关系
4. 滑坡谬误：夸大连锁反应

返回 JSON：
{
  "has_fallacy": true/false,
  "fallacy_type": "循环论证/幸存者偏差/因果倒置/滑坡谬误/无",
  "explanation": "详细说明",
  "severity": 0.8,
  "suggestion": "修改建议（如有）"
}
```

## 输出格式

```json
{
  "has_fallacy": true,
  "fallacy_type": "幸存者偏差",
  "explanation": "标题只关注了成功案例（富豪），忽略了失败案例（同样早起但不富的人）",
  "severity": 0.7,
  "suggestion": "修改为：为什么'部分'富豪都早起？"
}
```

## 严重程度阈值

| Severity | 处理 |
|----------|------|
| > 0.7 | 标注⚠，提供修改建议 |
| 0.5-0.7 | 标注⚠，不强制修改 |
| < 0.5 | 标记为通过 |

## 在最终输出中的展示

```
■ 候选标题
  ① [逆向拆解] 为什么 AI 让"提问"比"执行"更值钱？
     蓝海 | 逻辑✓
  ② [微观切片] 程序员用 ChatGPT 后反而更焦虑的真相
     黄海 | 逻辑⚠幸存者偏差
```

逻辑⚠表示有逻辑警告，后面的文字表示谬误类型。
